No, multiple users can ask questions simultaneously. The bot handles each `messageCreate` event independently and asynchronously.

Here's what actually happens:

- Each `@mention` triggers a separate `messageCreate` event
- Node.js processes them concurrently via async/await — no blocking
- Each event runs its own independent pipeline: embed question → search Supabase → call Gemini → reply

Because of how the `guild_settings` table is designed — it stores **one row per server**, and that row has a single `help_channel_id` column.

- Change `guild_settings` to a one-to-many relationship — a separate `guild_channels` table where each row is `(guild_id, channel_id)`

But for the current use case — one designated help/support channel per server — a single channel is intentional. It keeps the knowledge base focused. If you indexed every channel, you'd get noise from off-topic conversations polluting the answers.
